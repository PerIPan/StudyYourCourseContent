CLA Knowledgebase — Architecture Approaches

Choose how to build your RAG-powered study companion

A

Next.js + Vercel AI SDK + PostgreSQL (pgvector)

Recommended — Full-stack Next.js app deployed on Vercel. Uses pgvector for embeddings stored in your existing PostgreSQL (or Vercel Postgres). Claude API for answers. Clean chat UI with source references.

Pros

  • Native Vercel deployment — push and it's live
  • Vercel AI SDK handles streaming chat out of the box
  • pgvector is battle-tested for RAG at this scale
  • Single codebase (React + API routes)
  • Easy OpenWhisperer integration via API endpoint
  • You already have Vercel token configured

Cons

  • Needs Vercel Postgres addon (free tier: 256MB, enough for ~20 courses)
  • Slightly more setup than pure serverless
B

Next.js + Pinecone (Vector DB) + Vercel

Same Next.js frontend but uses Pinecone as a dedicated vector database instead of pgvector. Separates vector search from relational data.

Pros

  • Pinecone optimized specifically for vector search
  • Free tier: 100K vectors (plenty for course content)
  • No PostgreSQL dependency for deployment

Cons

  • Extra service to manage (Pinecone account + API key)
  • Adds vendor dependency
  • Slightly higher latency (external API call vs local DB)
C

Next.js + ChromaDB (self-hosted) + VPS

Open-source vector DB running on a VPS (e.g., Railway, Fly.io). Full control, no vendor lock-in, but more ops overhead.

Pros

  • Fully open-source stack
  • No vendor lock-in on vector storage
  • Rich Python ecosystem for document processing

Cons

  • Need separate hosting for ChromaDB (not serverless-friendly)
  • More infrastructure to maintain
  • Can't deploy cleanly on Vercel alone

My Recommendation: Option A

For your use case — class-shared, Vercel-deployed, with OpenWhisperer integration — Next.js + pgvector is the cleanest path. You already have Vercel configured, PostgreSQL experience, and it keeps everything in one stack. The ~23 documents you have will generate maybe 2-3K vector chunks — pgvector handles this trivially.
